Installation

Simply download the script, make it executable and run!

$ curl https://raw.github.com/zalando/system/master/install.sh
$ chmod +x install.sh
$ ./install.sh

The script will tell you which dependencies are missing, ask if you want to download and install them, get the latest stable source code and install required Node.js modules.

Manual installation

If you want to install it manually, please make sure you have installed:

  • Node.js (0.8.x +)
  • MongoDB (1.8.2 +)
  • ImageMagick (optional to generate map thumbnails)

Download the latest source code from GitHub:

$ git clone https://github.com/zalando/system.git ./

Configuring the server

All server settings are located on the file . If you want to override settings, please edit (or create) the file with your own values. Detailed instructions are available on the comments on the top of the file.

The following settings will certainly need your attention:

Settings.General
  • Sets the desired title for the app, something like "MyCompany System".
  • This should be set to false on production environments.
Settings.Database
  • MongoDB connection string. Default is host "localhost", database "systemapp".
Settings.Web
  • The port used by the Node.js server, default is 3003.
  • Set to true if you're deploying to common PaaS services. More info below.

Running the app

To start the System App, run the server.js file using node:

$ node server.js

This will start the Node.js server. If you haven't touched the settings, opening on your web browser should work. Otherwise replace the URL with your defined web port.

Production vs. Debugging

By default Node.js will run in "Debug mode". If you're deploying and running on production, you must set the to "production" to avoid having all the debugging statements logged to the console. Putting it simple:

$ NODE_ENV=production node server.js

Please note that the System App will enter debug mode automatically when it runs under the or a hostname. In debug mode, most actions and procedures will be logged with details so performance might be affected.

Running the server forever

If you want the server to run like a service (so it restarts itself in case of crash / error / termination) we recommend using the module. You can install it (globally) using NPM:

$ sudo npm install -g forever

To start the System App using forever, run it under the app root folder:

$ forever start -c node server.js
